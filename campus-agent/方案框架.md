# 校园数据导航 —— 多源数据融合驱动的学生服务智能体

> 赛道：智能体开发（自选题目）
> 参赛方向：数据采集 → 数据治理 → 数据分析 → 智能体开发 → 可视化 → 综合应用

---

## 一、项目概述

### 1.1 项目背景

高校学生在校期间需要频繁查询课表、成绩、空教室、一卡通消费、图书馆座位、校园通知等多类信息，但数据分散在教务系统、一卡通系统、图书馆系统、校园门户等互不联通的信息孤岛中，学生查找效率低下。

《"数据要素×"三年行动计划（2024-2026年）》明确提出推动教育领域数据融合应用。《提升全民数字素养与技能行动纲要》要求提升大学生数字素养。本项目以**校园多源数据融合**为底座，以**大语言模型智能Agent**为交互界面，实现"一句话查全校信息"的校园数据导航服务。

### 1.2 应用场景

- **课业查询**："明天上午在哪上课？"
- **消费分析**："我这个月食堂花了多少钱？和上月比呢？"
- **空教室查找**："现在图书馆还有空位吗？三教哪间教室没课？"
- **通知速览**："最近有什么竞赛报名？"
- **个性化推荐**："我这学期绩点3.5，适合报什么比赛？"

### 1.3 主要创新点

1. **多源异构数据实时融合**：首次将教务、消费、图书、通知四类校园数据统一接入Agent
2. **RAG + Function Calling 双层架构**：检索增强生成处理非结构化数据，函数调用处理结构化实时查询
3. **数据要素驱动的个性化推荐引擎**：基于学生行为数据画像实现精准推荐

### 2.2 核心功能模块

| 模块 | 功能 | 技术支持 |
|------|------|---------|
| 智能问答 | 自然语言查询校园数据 | RAG + LLM |
| 数据看板 | 个人消费/学习数据可视化 | ECharts |
| 空教室/座位 | 实时查询空闲资源 | Function Calling |
| 智能推荐 | 竞赛/活动个性化推荐 | 协同过滤 + LLM |
| 管理后台 | 数据导入、知识库管理 | Web Admin |

### 2.3 关键技术

- **RAG**：基于ChromaDB向量数据库，对校园通知、政策文件、常见问答做语义检索
- **Function Calling**：对课表、消费、空教室等结构化数据，用Agent的Tool机制实时查询API
- **多轮对话记忆**：维护用户Session，支持上下文关联追问
- **数据脱敏**：学生个人数据本地处理，不泄露隐私

## 三、应用价值

### 3.1 社会效益

- **提升校园数字化服务水平**：打通数据孤岛，一个入口解决所有查询需求
- **培养学生数据素养**：通过数据看板和消费分析，帮学生建立数据意识
- **可复制推广**：方案适用于国内绝大多数高校，标准化的数据接口设计便于推广

### 3.2 经济效益

- **降低信息获取时间成本**：每次查询从平均5分钟（多系统切换）降至10秒
- **减少重复咨询人工成本**：预计减少辅导员/教务处30%的日常咨询工作量

### 3.3 数据要素价值释放

- 将原本沉睡的校园数据（消费流水、借阅记录等）激活为可服务学生的**数据产品**
- 践行"数据要素×教育"的顶层设计，为高校数据治理提供可参考范式
